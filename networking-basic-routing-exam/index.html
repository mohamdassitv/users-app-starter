Exam: Configure Basic Routing

Scenario

You have three Linux machines on two networks:

  • leaf-01 â€” 192.168.178.10/24 on net-1
  • leaf-02 â€” 10.0.0.20/16 on net-2
  • router â€” dual-homed gateway: 192.168.178.2/24 (net-1) and 10.0.0.2/16 (net-2)

Nodes cannot reach each other yet because they lack proper routing.

Goal

Configure basic routing so that leaf-01 and leaf-02 can communicate via the router.

Hands-on Lab (Docker)

  1. Install Docker & Docker Compose.
  2. make up â€“ launches leaf-01, leaf-02, and router containers on two isolated networks.
  3. Open shells: make sh-leaf01, make sh-leaf02, make sh-router.
  4. Do the configuration (see Exam Tasks below).
  5. Verify with make grade.
Tip: reset the lab

Run make reset to recreate clean containers (it removes default routes on the leaves for you).

Exam Tasks

  1. On leaf-01, set the default route via the router on net-1.
    ip route del default || true; ip route add default via 192.168.178.2
  2. On leaf-02, set the default route via the router on net-2.
    ip route del default || true; ip route add default via 10.0.0.2
  3. On router, enable IPv4 forwarding.
    echo 1 > /proc/sys/net/ipv4/ip_forward
  4. Test connectivity.
    • From leaf-01: ping -c 3 10.0.0.20
    • From leaf-02: ping -c 3 192.168.178.10
